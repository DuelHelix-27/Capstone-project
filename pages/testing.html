Testing & Validation Plan

This section describes how Hidden In Plain Sight will be tested across the Android frontend, the native C++ backend, and full system workflows.

Unit Tests

Unit tests verify that individual components behave correctly before the full system is tested.

Component Test Expected Result
Kotlin UI / Logic Reject messages longer than 100 characters Input is blocked and an error is shown
C++ Steganography Layer Test a mock 8x8 DCT block with a binary input DCT coefficients are modified as expected

Integration Tests

Integration tests verify that the Android frontend, JNI bridge, and native C++ backend work together correctly.

JNI Embedding Flow

The frontend sends a test image buffer and a 50-character message through JNI, the C++ backend performs embedding, and the frontend receives the returned stego image buffer.

Expected Result: The returned image buffer is valid, with no memory leaks or data loss.

Hidden Unlock Flow

The disguised journal interface is tested to confirm that the correct 5-tap unlock sequence reveals the hidden application.

Expected Result: Correct gesture opens the hidden interface successfully.

Performance Benchmarks

Performance testing checks whether the application can run reliably on older Android devices under realistic field constraints.

Metric Target
Image Input 12MP JPEG reconnaissance image
Message Length 100 characters maximum
Embedding Time Under 30 seconds
System Stability No crash on older Android hardware
Resource Usage No critical CPU or RAM spikes
