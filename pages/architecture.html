Architecture Overview

Hidden In Plain Sight is a standalone Android application designed for covert communication in field environments. The system allows users to embed short hidden messages inside JPEG images and later extract them using the same application. Its architecture prioritizes offline operation, low runtime overhead, and minimal retention of sensitive data.

System Architecture Diagram

The diagram below illustrates the internal structure of the application, including the user-facing Android layers, the embedding and extraction logic, and the local image-processing workflow.

Architecture Layers

Camera Layer

Handles image capture or selection from local storage and passes the chosen image into the application workflow.

Android UI Layer

Allows the user to choose embed or extract mode, display the selected image, input a hidden message, and view extraction results.

Application Logic Layer

Validates message length and image type, coordinates embedding and extraction workflows, and ensures the correct processing path is used.

Steganography and Image Processing Layer

Performs compression-aware embedding and extraction using JPEG DCT coefficient modification, then rebuilds and saves the resulting image locally on the device.

Deployment Architecture

The application is intended for offline field use. While traditional civilian applications are often distributed through public app stores, this system is better suited for direct device deployment in controlled environments.

Security Architecture

Security Goals

  • Messages are concealed inside ordinary JPEG images.
  • All embedding and extraction occurs locally on the device.
  • Plaintext messages are not stored after embedding or extraction.
  • A gesture-based access trigger helps conceal the application interface.
  • The system avoids dependency on external servers or databases.

Known Limitations

  • It does not claim resistance against classified military-grade steganalysis.
  • If adversaries know the app is in use, extraction attempts become more likely.
  • The project focuses on covert messaging, not full enterprise key management.
